## Overview

Manual verification that the page renders correctly across three target viewports. This test covers image scaling, typography, and CTA button alignment.

---

## Target Viewports

| Label   | Width   | Device class       |
|---------|---------|--------------------|
| Mobile  | 375px   | iPhone SE / small phones |
| Tablet  | 768px   | iPad / mid-size tablets  |
| Desktop | 1280px  | Standard laptop/monitor  |

---

## Test Checklist

### 1. Layout & Structure

- [ ] Page content is fully visible at all three viewports — no horizontal scroll
- [ ] Sections stack vertically on mobile (375px) and tablet (768px)
- [ ] Multi-column grid switches to single-column on mobile
- [ ] No content is clipped or hidden unintentionally

### 2. Image Scaling

- [ ] Images scale proportionally and do not overflow their containers
- [ ] Images are not pixelated or over-stretched at 1280px
- [ ] Hero/banner image fills the intended width at each breakpoint
- [ ] `object-fit` behavior is correct (no squishing of aspect ratio)
- [ ] Lazy-loaded images appear when scrolled into view

### 3. Font Sizes & Typography

- [ ] Body text is legible at 375px (minimum 14px recommended)
- [ ] Headings (H1–H3) scale down gracefully on smaller viewports
- [ ] Line lengths do not become excessively wide on desktop (max ~75ch)
- [ ] No text overflow or truncation that hides meaningful content
- [ ] Font loading does not cause layout shift (FOUT/FOIT acceptable)

### 4. CTA Button Alignment

- [ ] CTA button is centered or aligned as designed at each viewport
- [ ] Touch target size is at least 44×44px on mobile (375px)
- [ ] Button text does not wrap or overflow the button boundary
- [ ] Button does not overlap adjacent elements at any viewport
- [ ] Hover and focus states are visible on desktop

### 5. Spacing & Padding

- [ ] Section padding reduces appropriately on mobile (no excessive whitespace)
- [ ] No collapsed margins cause visual stacking issues
- [ ] Footer and header do not overlap body content

---

## How to Run the Test

### Browser DevTools (recommended)

1. Open the page in **Chrome** or **Firefox**.
2. Open DevTools → Toggle Device Toolbar (`Ctrl+Shift+M` / `Cmd+Shift+M`).
3. Set viewport width manually to each target size: **375**, **768**, **1280**.
4. Walk through each checklist item above.
5. Repeat in **Safari** for WebKit rendering differences.

### Optional: Responsive screenshot tools

- [Responsively App](https://responsively.app/) — view all breakpoints simultaneously
- Chrome DevTools → "Responsive" preset → drag handle to each width

---

## Pass / Fail Criteria

| Severity | Condition |
|----------|-----------|
| **Fail** | Content is clipped, invisible, or overflows at any viewport |
| **Fail** | CTA button is misaligned or inaccessible on mobile |
| **Fail** | Images break layout (overflow, wrong aspect ratio) |
| **Warn** | Minor spacing inconsistency that does not affect usability |
| **Pass** | All checklist items confirmed across all three viewports |
