## Overview

Manually verify that the page renders correctly across three standard viewport widths. Checks cover image scaling, typography, and CTA button alignment.

---

## Test Viewports

| Label   | Width   | Device Target         |
|---------|---------|-----------------------|
| Mobile  | 375px   | iPhone SE / small phones |
| Tablet  | 768px   | iPad / medium screens |
| Desktop | 1280px  | Standard laptop/monitor |

---

## How to Set Viewport in Browser

### Chrome DevTools
1. Open DevTools (`F12` or `Cmd+Option+I`)
2. Click the **Toggle Device Toolbar** icon (`Ctrl+Shift+M` / `Cmd+Shift+M`)
3. Set a custom width in the dimensions field at the top

### Firefox Responsive Design Mode
1. Open DevTools (`F12`)
2. Click **Responsive Design Mode** (`Ctrl+Shift+M`)
3. Enter the target width manually

## What to Look For

### Image Scaling
- Images should use `max-width: 100%` to prevent overflow
- Aspect ratio must be preserved (no squashing or stretching)
- Hero/banner images should cover their container without cropping important content

### Font Sizes
- Body text should remain legible at all viewports
- Heading hierarchy (h1 > h2 > h3) must be visually distinct
- Line-height should not cause text to overlap

### CTA Button Alignment
- Must be clearly visible above the fold on mobile
- Should not be pushed off-screen or obscured by other elements
- Touch target must be large enough on mobile (â‰Ą 44Ă—44px)

## Pass / Fail Criteria

| Check | Pass | Fail |
|-------|------|------|
| Horizontal scroll | Not present | Scroll appears at any viewport |
| Image distortion | None | Aspect ratio broken |
| Font legibility | All text readable | Any text too small or overlapping |
| CTA visibility | Fully visible and clickable | Hidden, clipped, or misaligned |
| Layout consistency | Matches design intent | Broken grid or overlapping elements |
